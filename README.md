## Deploying to Render

The repo ships with a complete `render.yaml` for one-click Infrastructure-as-Code deployment.

### Services created automatically

| Service | Type | Plan |
|---|---|---|
| `cardvault` | Web (Gunicorn) | Free |
| `cardvault-worker` | Background worker (Celery) | Free |
| `cardvault-beat` | Background worker (Celery Beat) | Free |
| `cardvault-redis` | Redis | Free |
| `cardvault-db` | PostgreSQL | Free |

### Steps

1. **Fork / push** this repo to GitHub.
2. In [Render Dashboard](https://dashboard.render.com) â†’ **New â†’ Blueprint** â†’ connect your repo.
3. Render reads `render.yaml` and provisions all services.
4. **Set secrets manually** in the Render dashboard for the `cardvault` web service:
   - `JUSTTCG_API_KEY` â€” your JustTCG API key
   - `EMAIL_BACKEND` â€” e.g. `django.core.mail.backends.smtp.EmailBackend` for real email
   - `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_HOST_USER`, `EMAIL_HOST_PASSWORD` â€” SMTP details
   - `DEFAULT_FROM_EMAIL` â€” e.g. `CardVault <hello@yourdomain.com>`
5. Render runs `python manage.py migrate` automatically on each deploy (see `buildCommand` in `render.yaml`).
6. Trigger a **manual deploy** once secrets are set.

### Build & Start commands (if setting up manually)

```
Build:  pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate --noinput
Start:  gunicorn cardvault.wsgi --workers 2 --threads 2 --log-file -
```

### Environment variables reference

| Variable | Required | Notes |
|---|---|---|
| `SECRET_KEY` | Yes | Auto-generated by Render Blueprint |
| `DATABASE_URL` | Yes | Auto-set from Postgres service |
| `REDIS_URL` | Yes | Auto-set from Redis service |
| `DEBUG` | Yes | Set to `False` in prod |
| `ALLOWED_HOSTS` | Yes | `.onrender.com` or your custom domain |
| `JUSTTCG_API_KEY` | Recommended | Card pricing; search falls back to local DB without it |
| `EMAIL_BACKEND` | No | Defaults to console; set SMTP for real alerts |
| `DEFAULT_FROM_EMAIL` | No | Sender address for price-surge emails |
